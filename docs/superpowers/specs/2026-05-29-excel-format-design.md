# Спека — формат Excel-книги master_all.xlsx (этап 1)

Дата: 2026-05-29
Файл реализации: `_extracted/crimea_parser/utils/excel_export.py`

## Цель

Привести выгружаемую книгу `master_all.xlsx` к читаемому, «продажному» виду: общий список + вкладки по крупным городам + сборник мелких, с осмысленной сортировкой строк. Структура колонок фиксируется как контракт под будущую выгрузку в Google Таблицу для агента-рассыльщика.

## Структура книги (порядок листов)

1. **`Сводка`** — статистика (всего, % с email/phone/адресом/сайтом, разбивка по источникам / типам клиентов / городам). Без изменений.
2. **`Крым`** — общий полный список ВСЕХ записей. (Переименование текущего листа `Все`.)
3. **Вкладки городов** — по одной на каждый город с **≥5 записей**, в порядке убывания количества (Ялта, Феодосия, Евпатория…).
4. **`Остальные`** — записи городов с **<5 адресов**, собраны вместе, чтобы не выискивать их в большом «Крым».
5. **`Без контактов`** — записи без phone и без email (для ручного добивания). Без изменений.

Дублирование ожидаемо и допустимо: каждая запись присутствует в «Крым» (общий) и дополнительно либо в своей городской вкладке (≥5), либо в «Остальные» (<5).

## Сортировка строк (внутри каждого листа)

Ключ сортировки (по убыванию приоритета):
1. есть email — выше (приоритет для рассылки);
2. есть телефон — выше;
3. название объекта — по алфавиту (А→Я).

Применяется ко всем листам с данными («Крым», городские вкладки, «Остальные», «Без контактов»).

## Форматирование (сохраняем как есть)

- Заморозка шапки, AutoFilter, ширина колонок по содержимому.
- Перенос текста в колонках «Адрес», «Комментарий».
- Заливка строки по типу клиента; светло-красная заливка строк без контактов.
- Кликабельные ссылки: телефон (`tel:`), email (`mailto:`), сайт, соцсеть.
- Цветные бейджи источника.

## Колонки (контракт под Google Sheets)

`Город · Название · Тип клиента · Категория · Адрес · Телефон · Email · Сайт · Соцсеть · Комментарий · Источник · Собрано`

Плоская таблица, объединённых ячеек нет — импорт в Google Таблицы пройдёт без потерь. Порядок и набор колонок не меняем без согласования (будущий агент-рассыльщик читает по этому контракту).

## Изменения к текущему коду

2. Константа `MIN_CITY_ROWS`: `10` → `5`.
3. Лист `Остальные` — оставить (уже реализован), порог <5.
4. Добавить сортировку строк по ключу выше во всех листах данных.
5. Порядок создания листов: `Сводка`, `Крым`, города (≥5), `Остальные`, `Без контактов`.
