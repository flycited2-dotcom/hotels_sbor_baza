# Спека — чистка VK-шума (этап 4)

Дата: 2026-05-30
Источник проблемы: `parsers/vk_groups.py` искал по матрице город × ключевое слово
(«база отдыха», «вилла», «апартаменты» и т.п.) — выдача VK цепляла турфирмы,
кафе, чаты, магазины с подходящими словами в названии/описании. Из ~6 245 VK-записей
в `master_all.csv` значительная доля была не отелями.

## Решение

Классификатор по двум полям (`name`, `activity`) с тремя метками:

| label | действие |
|---|---|
| `hotel` | есть отельный стем → принимаем |
| `noise` | отельных нет, есть шумовой → отбрасываем |
| `ambiguous` | ни тех, ни других → сохраняем, помечаем `comment="vk_review"` |

Ключевые слова — стемы (`отел`, `гостиниц`, `санатор`, `пансионат`, `размещен`,
`апарт`, `мотел`, `усадьб`, …) + точные формы (`коттедж`, `глэмпинг`, `база отдыха`,
`вилла`, `виллы`). Шум: коммерция/услуги/товары, транспорт-прокат, event (свадьбы,
банкеты, рестораны, кафе), фитнес/здоровье/красота, образование, туризм-агентства,
бытовка, коммьюнити (чат, объявления, юмор), отдельно — частые noise-марки
в названиях («тент-мастер», «клининг», «уборка», …).

## Файлы

- `parsers/vk_filter.py` — `HOTEL_KEYWORDS`, `NOISE_KEYWORDS`, `classify(name, activity)`.
- `parsers/vk_groups.py` — врезан `vk_classify` перед `save_item`: noise пропускается,
  ambiguous сохраняется с `comment="vk_review"`. Финальный лог:
  `[VK] добавлено: N | пропущено шума: M | помечено ambiguous: K`.
- `cleanup_vk.py` — retro-чистка существующего `master_all.csv`. По умолчанию
  dry-run (статистика + 15 примеров). С `--apply`:
  - backup `master_all.csv.bak.<timestamp>`,
  - перезапись `master_all.csv` без noise,
  - `output/master_all_noise.csv` со снятыми записями (аудит/восстановление).
