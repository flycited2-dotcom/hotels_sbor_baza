## 1. Цель

Регулярно рассылать персональные коммерческие предложения по email владельцам/менеджерам
объектов размещения в Крыму (отели, пансионаты, базы и дома отдыха, эллинги) от имени
гендиректора «Технолайн Трейд», получать обратную связь и контролировать воронку, чтобы
ни один лид не потерялся.

Пилот за 1–2 недели на ~500 контактах показывает: работает ли схема, какова доля ответов,
не блокируется ли почта — и нужно ли строить полноценную систему.

## 6. ТЗ: «Claude в Chrome» — разовая настройка почты

**ЦЕЛЬ:** настроить SPF/DKIM/DMARC для `simfer.com.ru` и связать ящик
`alexey.gurinenko@simfer.com.ru` (SprintHost) с Gmail `flycited2@gmail.com`.

**ПРАВИЛА БЕЗОПАСНОСТИ:** перед сохранением каждой DNS-записи показать её пользователю и
дождаться подтверждения; существующие записи не удалять; после изменений — проверить валидатором.

### Часть 1 — DNS (панель SprintHost)
1. Войти в панель SprintHost → домен `simfer.com.ru` → DNS-зона. Проверить, что DNS управляется
   здесь (NS → SprintHost); если нет — сообщить и остановиться.
2. **SPF:** при отсутствии добавить рекомендованную SprintHost (обычно
   `v=spf1 +a +mx include:_spf.sprinthost.ru ~all` — сверить с подсказкой SprintHost);
   существующий SPF не дублировать.
3. **DKIM:** включить в разделе почты; убедиться, что запись появилась в зоне.
4. **DMARC:** TXT, хост `_dmarc.simfer.com.ru`, значение
   `v=DMARC1; p=none; rua=mailto:alexey.gurinenko@simfer.com.ru; fo=1`.
5. Показать итог изменений.

### Часть 2 — Связка с Gmail (`flycited2@gmail.com`)
6. Gmail → Настройки → «Аккаунты и импорт».
7. «Отправлять письма как» → `alexey.gurinenko@simfer.com.ru`, имя «Алексей Гуриненко»;
   SMTP `smtp.simfer.com.ru`, порт 465 (SSL) [или 587 TLS], логин = полный адрес, пароль из менеджера.
8. Подтвердить код: открыть веб-почту SprintHost, найти письмо Gmail, ввести код/ссылку.
9. (опц.) Сделать адрес отправкой по умолчанию.
10. Приём: «Получать почту из других аккаунтов (POP3)» → `mail.simfer.com.ru`, порт 995 (SSL),
    логин/пароль. (Альтернатива — пересылка в веб-почте SprintHost на `flycited2@gmail.com`.)

### Часть 3 — Проверка
11. Тест из Gmail «от» домена на mail-tester.com → SPF/DKIM/DMARC = pass, «from» домена (не «via gmail»).
12. Ответ извне → проверить, что пришёл в `flycited2@gmail.com`.
13. Финальный отчёт.

## 7. Выполнение кампании — в Cowork (Gmail-коннектор)

После настройки почты сама рассылка ведётся в Cowork:
1. Прочитать из Google-таблицы следующие N контактов со Статус = «новый».
2. Сгенерировать персональные письма по шаблону (тема — по ротации).
3. Режим A: создать черновики в Gmail «от» домена → пользователь проверяет и шлёт.
   Режим B (по разрешению): отправлять с паузой 30–60 сек.
4. Обновить статус/дату в таблице.
5. Раз в день разобрать ответы: классифицировать (интерес/запрос КП/отказ), пометить,
   выдать список «требует реакции». Контрольная точка: «запрос КП» старше 24 ч без ответа.
6. Фоллоу-ап молчунам через 5 дней (другой текст), пометка «фоллоу-ап + дата».

**Лимиты/прогрев:** дни 1–3 → 10–15/день, дни 4–7 → 20–30, далее → до 50; не писать
«ответил/отказ/готово»; уважать отписку.

## 8. Честные ограничения и риски

- **Браузерный агент (Chrome)** — только для настройки; для кампании не используется (медленно/хрупко).
- **Шейред-хостинг SprintHost** = общий IP и лимиты; при плохой доставляемости — отдельный домен/сервис.
- **Доставляемость холодных писем:** обязательны SPF/DKIM/DMARC, дрип, строка-отписка, мониторинг жалоб.
- **Безопасность:** пароль засветился в переписке — сменить после настройки; в файлах не хранить.

## 10. Критерии успеха пилота (1–2 недели)

- Пройдены первые ~150–250 контактов без блокировки почты.
- Понятна доля ответов (reply rate).
- Ни один входящий запрос не потерян.
- Решение: достаточно ли схемы или строим полноценную систему и переходим к CRM.
